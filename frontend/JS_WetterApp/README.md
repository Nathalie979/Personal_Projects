# Wetter App – Frontend Projekt

## Projektkontext
Dieses Projekt ist ein **Frontend-Übungsprojekt** zur Anwendung von **HTML, CSS und JavaScript**.  
Ziel war die Umsetzung einer einfachen Wetterabfrage mit Fokus auf **Formularverarbeitung, DOM-Manipulation und visuelle Darstellung**.

Es handelt sich **nicht** um eine produktive oder vollständig ausgebaute Anwendung.

---

## Aufgabenstellung
Entwicklung einer Webanwendung, bei der Nutzer:innen:
- eine Stadt und/oder Postleitzahl eingeben können
- Wetterinformationen angezeigt bekommen
- bei ungültigen Eingaben eine Fehlermeldung erhalten

Der Schwerpunkt lag auf Funktionalität, Benutzerfeedback und sauberem Frontend-Code.

---

## Projektziel
- Verarbeitung von User-Input über ein Formular
- Validierung von Eingaben
- Dynamische Anzeige von Inhalten
- Trennung von Struktur, Logik und Styling
- Responsive Darstellung

---

## Meine Leistungen

### Funktionalität
- Formular zur Eingabe von Stadt und Postleitzahl
- JavaScript-Funktion zur Verarbeitung der Suche
- Dynamisches Ein- und Ausblenden von Informations- und Fehlerzuständen
- Anzeige von Wetterdaten (Text + Icon)

---

### Frontend & Styling
- modernes, reduziertes UI
- Einsatz von CSS Custom Properties
- visuelle Hervorhebung von Eingabe- und Ergebnisbereichen
- klare Fehlermeldungen bei ungültiger Eingabe
- responsive Anpassung für mobile Geräte

---

## Technischer Fokus
- HTML5
- CSS (Layout, Variablen, Responsiveness)
- Vanilla JavaScript
- DOM-Manipulation
- Formular-Handling
- einfache Fehlerbehandlung

---

## Projektstruktur
- `index.html` – Struktur der Anwendung
- `style.css` – Layout, Farben und Responsiveness
- `script.js` – Logik für Suche, Anzeige und Fehlerzustände
- `icons` – enthält die notwendigen Icons

---

## Hinweis zur Einordnung
Dieses Projekt ist eine **Frontend-Übung** und dient der Festigung grundlegender Webtechnologien.  
Es ist **keine vollständige Wetterplattform** und enthält **keine serverseitige Logik**.
